## Minor Issues

### 1. `MyCertificationsPage` — `length > 0` assertions where text legitimately duplicates

Lines 237, 244-246, 266: `getAllByText('CPR Certification').length > 0`
and `findAllByText('Valid').length > 0`. The author documented (line 236
comment) that "CPR Certification" appears in both the required-certs
section and the action-required section — so an exact count couples the
test to layout choices it doesn't care about. The `length > 0` check is
defensible here; it just doesn't *prove* both sections render the cert.
A tighter version would scope per-section: `within(requiredSection)
.getByText('CPR Certification')` and the same for `actionSection`.
Non-blocking.

### 2. `MyCertificationsPage` — direct `api.get` mock vs. service layer

Author flagged this. Confirmed by reading `MyCertificationsPage.tsx`:
the page calls `api.get('/teacher/certifications/').then(r => r.data)`
inline — no service module. Mocking `api.get` is therefore correct for
*this* page. Worth a backend-or-frontend follow-up to extract a
`teacherCertificationsService` so future tests can mock at the same
layer everyone else does. Not a test-side issue.

### 3. Test count discrepancy on FE-053

The agent recount surfaced 31 `it(` blocks in `MyClassesPage.test.tsx`
versus the author's claim of 26. Spot-check confirms the file is well-
structured; the higher number is just additional small assertions split
into discrete `it()` blocks. Author should re-count for the next
review request, but the work is sound either way.

## Positive Observations

- **Class-Teacher badge conditional rendering** (shown for
  `is_class_teacher: true`, absent for `false`) — exact kind of boolean
  conditional that often regresses without coverage.
- **Singular-vs-plural copy** ("Total Section" vs. "Total Sections",
  "Subject" vs. "Subjects") tested explicitly.
- **Expand/collapse interaction** on cert cards is fully exercised:
  click → labels appear → URL link's `href` asserted → click again →
  collapses.
- **Optional fields handled**: `department` badge shown when present,
  hidden when null; `grade_band_name` subtitle optional.
- **`uppercase` CSS class vs. DOM text-case caveat** documented (line
  ~49 of the request) — author correctly asserted the title-case DOM
  text, not the visually-rendered ALL CAPS.
- **Compliance percentage computation** (75% from
  `required_met / required_total`) verified against fixture math.
- **Empty-state texts** distinct from error-state texts — both covered.

## Recommendations (non-blocking)

- For FE-054: scope `getAllByText` calls inside section containers
  using `within()` so the test proves both sections render the same
  cert string.
- Consider extracting `teacherCertificationsService` so the page
  matches the service-layer pattern used by sibling teacher pages
  (separate task — not a test concern).
- Reconcile FE-053's claimed test count (26) with the actual count (31)
  in future review requests.
