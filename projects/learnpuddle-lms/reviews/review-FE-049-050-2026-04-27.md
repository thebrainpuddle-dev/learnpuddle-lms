# Review: FE-049 (CourseTemplateGalleryPage) + FE-050 (SectionDetailPage)

## Verdict: APPROVE

## Summary

49 new tests across the platform course-template browse page and the
section-detail tabbed view. 1233/1233 full-suite green. Both suites
exercise filter UI, tab navigation, modal flows, and TanStack Query
loading/error/empty states.

## Critical Issues

None.

## Major Issues

None.

## Minor Issues

### 1. `SectionDetailPage` — placeholder-text selector

Line ~475: `getByPlaceholderText('Search students by name, email, or ID...')`.
A small copy change in the placeholder would silently break this test.
Replacing with `getByRole('searchbox')` or a `data-testid` on the input
would be more durable. Style nit; non-blocking.

### 2. `SectionDetailPage` — `getByTitle('Back to grade')`

Line ~618. `title=` attributes are easy to drop in refactors and don't
contribute to accessibility. Prefer `getByRole('link', { name: /back/i })`.

### 3. `CourseTemplateGalleryPage` — child components stubbed as testid divs

`TemplateCard`, `TemplatePreviewPanel`, `CloneTemplateDialog`, `EmptyState`
all stubbed. The stubs render the data they receive (template title in
the card stub, `role="status"` on the empty stub) so the parent
behaviour under test — filter/search wiring, click-to-preview, count
rendering — is exercised against real data. Stubbing is justified for
isolation; the stubbed components have their own unit tests elsewhere.

### 4. `SectionDetailPage` — search test relies on debounce timeout

Line ~485-492: `{ timeout: 2000 }` waits for the debounce. Works, but
makes this test slower. If the debounce delay is exposed as a config
constant, a `vi.useFakeTimers()` + `vi.advanceTimersByTime(N)` pattern
would be deterministic and faster. Non-blocking.

## Positive Observations

- **Server-side vs. client-side filter distinction** in
  CourseTemplateGalleryPage: search is asserted client-side
  (`/IB PYP/`), while category/language dropdown changes are asserted
  to call `listTemplates({ category: 'TEACHING_SKILLS' })` —
  correctly mirrors the page's split filter strategy.
- **Singular-vs-plural count test** (`"1 template found"` vs.
  `"2 templates found"`) — the kind of i18n-adjacent edge case that
  always regresses without coverage.
- **Tab navigation in SectionDetailPage** covers Students → Teachers →
  Courses with mock-driven content for each.
- **Author noticed `TeachingAssignment` shape** for
  `SectionTeachersResponse.teachers` and updated fixtures — that's the
  kind of investigation that prevents shipped-but-broken integration.
- **Per-tab error-state coverage** — failed teachers query, failed
  courses query, failed students query — each independently covered.
- **Modal validation** (`First name is required`) tested on the
  empty-submit happy/error edges.

## Recommendations (non-blocking)

- Replace `getByPlaceholderText` and `getByTitle` selectors with
  role-based queries.
- Consider fake timers for the debounce-search assertion.
