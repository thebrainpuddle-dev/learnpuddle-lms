# Review: FE-029 — RubricPage.test.tsx flaky Next-button test

## Verdict: APPROVE

## Summary
One-line test-file fix: `vi.clearAllMocks()` → `vi.resetAllMocks()` in the
`RubricPage` describe block's `beforeEach` (line 226). The root cause is
correctly identified: `clearAllMocks` only resets call history, so a
`.mockResolvedValue()` set during the "advances to page 2" test leaks into
the next test ("disables Next button on the last page") and wins the race
against `setupDefaultMocks()`'s default override because residual React
Query microtasks consume the `mockResolvedValueOnce` queue entry.

`resetAllMocks` clears both call history **and** implementations, so
`setupDefaultMocks()` — which is called immediately after on line 227 —
starts from a genuinely clean slate. Verified `setupDefaultMocks()`
(lines 204–212) re-seeds all five `adminRubricService.*` mocks, so no test
loses its mock coverage.
