## Minor Issues

1. **Loading skeleton queried by class** (`document.querySelector('.animate-pulse')`, L224).
   Same coupling concern flagged in FE-059-063 and FE-064-068. Recommend
   `data-testid="loading-skeleton"` on the production component for stability. Non-blocking.

2. **`getByTitle('Delete')` is intentionally generic** (L308, L318). The `title="Delete"`
   attribute on the production component (L292) is also a tooltip, not a screen-reader-
   accessible label. **Production a11y issue, not a test issue** — file as a separate
   FE-a11y ticket: replace `title="Delete"` with `aria-label="Delete certification type"`.
   Test workaround is fine.

3. **`screen.getByText('Yes')` and `screen.getByText('No')`** (L263, L270). These are short
   tokens that *will* collide in a tab with multiple boolean badges or radio groups. Today the
   fixture renders only one row, so they're unique — but tests should make their scoping
   intent explicit (e.g., `within(row).getByText('Yes')`) to survive future fixtures with
   multiple types.

4. **`getByRole('button', { name: /create/i })`** for the modal submit (L296) will collide with
   the modal's `Cancel` button if anyone retitles it to "Cancel and create new" or similar.
   Loose anchor; `/^create$/i` would tighten it.

5. **URL-param test asserts only stub rendering, not panel transition.** `?tab=approvals`
   tests confirm the stub appears (L330) but don't verify the default Certification Types
   panel is hidden. A `.queryByText(/no certification types defined yet/i)` not-in-document
   assertion would catch a regression where both panels render simultaneously.

## Positive Observations

- **All 6 heavy sub-tabs stubbed** keeps this test file fast and focused on the page-level
  routing/render concerns. The right scope.
- **URL-param routing tested via `MemoryRouter initialEntries`** — exactly the right
  pattern for `useSearchParams()`-driven tabs. Better than mocking the router hook directly,
  because the actual tab-index computation is exercised.
- **Create-modal submission asserts on payload shape** (`toMatchObject({ name: 'CPR Training' })`,
  L298), not just call count. Catches a real "form sends empty/wrong values" regression class.
- **Delete-confirm path covers both the dialog opening AND the API call after Confirm**
  — both halves of a destructive-action flow.
- **`beforeEach` sets sensible empty-state defaults for all three sub-tabs' services** so each
  test only overrides what it cares about. Idiomatic.
- **`expiryCheck` mock returns the full shape** (`{ expiring_soon, already_expired,
  threshold_days }`) — guards against future TS refactors that add field requirements.
- **`certificationsService.types.{list,create,update,delete}`** namespacing in the mock matches
  the production service exactly — fewer surprises if the service is later destructured at
  the call site.
