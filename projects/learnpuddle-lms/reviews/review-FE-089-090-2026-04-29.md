## Security-sensitive verifications

### SSOCallbackPage — token storage boundary ✅
Lines 163-183 verify both tokens land in **`sessionStorage`** (not `localStorage`):
- `sessionStorage.getItem('access_token')` ✅
- `sessionStorage.getItem('refresh_token')` ✅

This is the correct boundary per the LMS auth spec (CLAUDE.md: "Stored in `sessionStorage` (session) or `localStorage` (remember me)"). SSO callback is a session-only flow, so sessionStorage is correct.

### SuperAdminLoginPage — role guard ✅
Lines 204-227 verify non-SUPER_ADMIN role:
- `expect(mockNavigate).not.toHaveBeenCalled()` (line 214) — no nav happens
- `expect(mockSetAuth).not.toHaveBeenCalled()` (line 227) — no auth state set
- Error message "This portal is for platform administrators only." surfaced
- Also asserts `portal: 'super_admin'` is sent in the login payload — guards against accidental mixed-portal logins

### ParentVerifyPage — magic link verification ✅
Lines 134-163 verify success-path triple:
- `verifyToken('magic-abc-123')` (line 138)
- `setSession()` with session/refresh tokens + children (line 146)
- `navigate('/parent/dashboard', { replace: true })` (line 162) — `replace: true` is critical (prevents back-button re-trigger)

### AcceptInvitationPage — email field locked ✅
Line 185 asserts the email input is `disabled` — the invitee cannot edit the address the invitation was issued to. Right boundary.
