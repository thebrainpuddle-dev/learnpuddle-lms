# Review: FE-034 — Wire Analytics Charts to Real Backend APIs

## Verdict: APPROVE

## Summary

Three analytics chart components (`DeadlineAdherenceChart`, `ApprovalTrendsChart`, `CourseEffectivenessChart`) are migrated from hardcoded `MOCK_DATA` to live `useQuery` calls against `adminReportsService`. New typed interfaces (`DeadlineAdherencePoint`, `ApprovalTrendsPoint`, `CourseEffectivenessItem`, `AnalyticsPeriodParams`) and three service methods land cleanly. Loading/error/empty states are present in every chart, matching the `CertComplianceChart` precedent. tsc clean, 619/619 vitest pass. The companion backend endpoints don't exist yet — qa-tester has already shipped 35 TDD tests defining the contract (separately approved 2026-04-24).

## Critical Issues

None.

## Major Issues

None.

## Minor Issues

1. **Stat-vs-error inconsistency in `DeadlineAdherenceChart`**
   `frontend/src/components/analytics/DeadlineAdherenceChart.tsx:78` renders `${latest?.adherencePercent ?? 0}%` whenever `isLoading` is false. If the request errored, the stat will display `"0%"` while the chart area shows `"Failed to load deadline data"`. Cosmetic but confusing. Suggest `isError ? '—' : `${latest?.adherencePercent ?? 0}%``. Same pattern in `ApprovalTrendsChart:72-76` (will display "0% overall approval rate (0 total requests)" on error). Not blocking.

2. **`CustomTooltip` typed as `any`**
   `DeadlineAdherenceChart.tsx:20` and `CourseEffectivenessChart.tsx:32`. Recharts' tooltip prop typing is genuinely awkward (`TooltipProps<number, string>` from `recharts`), so `any` is pragmatic and matches the existing codebase. Worth filing a one-shot follow-up to standardize on a `RechartsTooltipProps` shared helper if it gets reused a fourth time.

3. **`adminReportsService.deadlineAdherence` / `approvalTrends` strip `undefined` keys but `courseEffectiveness` takes no params**
   Asymmetry is intentional and documented in the cover note ("date-range filtering can be wired later without a service change"). Fine.

4. **Stale-time of 5 minutes**
   Matches `CertComplianceChart`. Reasonable. Note that until backend endpoints exist, the user will see the error state for 5 minutes per dashboard visit before retrying. Acceptable during the BE→FE handoff window.

5. **No test additions for the new chart wiring**
   The cover note says "no regression in passing tests" but doesn't mention new tests for the three charts. The `useQuery` integration is simple enough that the existing component tests likely still cover the render branches via mock service responses. Confirm that `frontend/src/components/analytics/*.test.tsx` (if they exist) were updated to mock `adminReportsService` rather than relying on the removed `MOCK_DATA`.

## Positive Observations

- **Strict typing**: every new interface is concretely typed; `useQuery<DeadlineAdherencePoint[]>` flows through to `data: DeadlineAdherencePoint[]`. No leaky `any` on the data path.
- **`rawData ?? []` pattern**: avoids conditional guards downstream, keeps `chartData` total. Same pattern across all three charts — DRY.
- **Per-chart spinner colors**: emerald/amber/purple match the heading icons. Polish that signals "this is still loading the same widget you clicked on," not a generic indeterminate state.
- **Approval rate "—" during loading**: `ApprovalTrendsChart.tsx:72-76` correctly avoids the "0 total requests" flash by hiding the parenthetical until data lands.
- **Service `clean` object pattern**: three new methods follow `engagementHeatmap`'s `if (params.x) clean.x = params.x` pattern. No `?start=undefined` strings sent to the wire.
- **Backward-compatible empty state**: pre-existing empty-state branches (`No deadline data yet`, etc.) are reachable for both "API returns []" and "first render before fetch" — graceful handoff.
- **TDD coordination**: this work paired with qa-tester's 35 TDD tests for the backend means backend-engineer can implement against an unambiguous contract. Clean two-team handoff.
