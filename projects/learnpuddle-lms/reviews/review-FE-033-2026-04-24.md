# Review: FE-033 — QuestionBankPage.test.tsx

## Minor Issues

1. **Time-based wait in MCQ-no-correct test**
   `frontend/src/pages/admin/QuestionBankPage.test.tsx:607` uses `await new Promise(r => setTimeout(r, 100))` to give RHF time to validate before asserting `svc.createQuestion` was not called. Magic-number sleeps are brittle on slow CI. Prefer `await waitFor(() => expect(svc.createQuestion).not.toHaveBeenCalled())` or `findByText` on whatever DOM artifact validation produces. Won't block but flag for the next sweep.

2. **Test count drift between cover note and actual file**
   The cover note claims "4 tests" for the Bank list section; the file ships 6 (heading, loading, populated, empty, search-specific empty, search-term). Total still equals 29. Not a defect — just keep cover notes synced for future review velocity.

3. **`getTypeSelect()` relies on combobox ordering**
   `within(dialog).getAllByRole('combobox')[0]` is correct today (only the type selector is a combobox in that modal) but a regression that adds a second combobox above it would silently retarget the first call. A `<label htmlFor="question-type">…</label>` plus `id="question-type"` on the select would make this rock-solid; the test note already documents the workaround.

4. **`Zod` superRefine error-path silence**
   Acknowledged in the cover note: when no MCQ choice is marked correct, the validation error fires at the `choices` path but the JSX doesn't surface it. The test correctly asserts behavior (`createQuestion` not called) rather than DOM text. Suggest filing a tiny follow-up for `QuestionBankPage.tsx` to render `form.formState.errors.choices?.message` so users get explicit feedback.

## Positive Observations

- **Dialog scoping discipline**: every confirm-dialog interaction uses `within(screen.getByRole('dialog'))` — eliminates the classic "two `Delete` buttons match" footgun.
- **Behavior-based assertions**: where DOM signal is absent, tests assert on side effects (`svc.createQuestion`, `svc.deleteBank`) rather than implementation internals.
- **DataTable mock is the right shape**: surfaces `data-field="…"` spans and invokes `cell()` for action columns. Sidesteps TanStack's render tree without losing the row-action coverage. Reusable pattern.
- **Mode coverage**: tests exercise create vs edit by asserting on `Create Bank` vs `Save Changes` button labels and pre-fill state — proves the modal mounts in both modes correctly.
- **Type-switching tests**: TRUE_FALSE/ESSAY/SHORT all tested for visibility of the choices section. Catches a class of regressions that pure unit tests on the form state would miss.
- **Empty-state taxonomy**: distinguishes "no banks yet" from "no banks match your search" — proves the UI can communicate both root causes.
- **No regressions**: 619/619 passing including the 29 new tests.

## Recommendation

Approve and merge. File two follow-ups for QA/FE polish:
1. Replace the 100ms sleep with `waitFor` (~5 min change).
2. Surface `choices` superRefine error in `QuestionBankPage.tsx` so the validation has a visible signal.
