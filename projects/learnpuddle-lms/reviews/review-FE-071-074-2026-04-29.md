### Anti-pattern scan (clean)
- No `it.skip` / `xit` / `it.todo`
- No bare `expect(true).toBe(true)` fake-pass
- No `as any` in test setup
- All `it()` blocks contain assertions

### Architecture verification
- All four files configure `QueryClient` with `staleTime: Infinity, retry: false, refetchOnWindowFocus: false` — matches the FE-056 stabilization pattern.
- `SchoolsPage` correctly wraps in `ToastProvider` (required for `useToast()` mutations).
- Mock schema `{ count, results }` matches API expectations for paginated endpoints.

## Minor Issues (non-blocking)
- `SchoolsPage` desktop/mobile dual-render scoping via `document.querySelector('.hidden.md\\:block')` is brittle to Tailwind class refactors. Consider adding a `data-testid="schools-table-desktop"` to the parent and querying by that. Not blocking; document the dependency in a comment if you keep the class selector.

## Positive Observations
- SchoolsPage covers the full bulk-email flow including modal open/close/cancel, payload shape, and post-success modal close — exactly the kind of multi-step UX that breaks silently.
- SuperAdmin DashboardPage tests `getStats` is called exactly once on mount — guards against StrictMode double-invoke regressions.
- Empty-state coverage is consistent across all four files (each tests both "no data" and "no matching filter" variants where applicable).
- Test count slightly under-claimed rather than over-claimed — refreshing.
