# Review: FE-045 (BillingPage) + FE-046 (CreateTeacherPage) — Page Test Suites

## Verdict: APPROVE

## Summary

49 new Vitest tests across two admin pages. All pass individually; full
suite is 1136/1136 green. Test-only changes — no production code touched.
Both suites use module-level service mocks, semantic role/text queries,
and cover happy + error + empty paths. Minor polish notes below; none
gate merge.

## Critical Issues

None.

## Major Issues

None.

## Minor Issues

### 1. `BillingPage.test.tsx` — currency regex uses unescaped dot

Lines 353, 363, 411: `/1.999/`, `/4.999/`, `/5.898/`. The `.` matches
*any* single character — works fine today because en-IN locale always
inserts a comma at that position, but would silently match unexpected
strings (e.g. `1Z999`) if upstream formatting changes. Not a current
defect. Tightening to `/[\d,]\s*1[,. ]999/` or `/₹\s*1[,. ]999/` would
be more deliberate.

### 2. `BillingPage.test.tsx` — selector via `document.querySelector`

Line 217 reaches outside `screen` for `.animate-spin`. Pre-existing
pattern in this codebase, but a `data-testid="loading-spinner"` on the
spinner element would be cleaner. Non-blocking.

### 3. `CreateTeacherPage.test.tsx` — error message assertion uses `expect.any(String)`

Line 365 — `toast.error(expect.any(String), ...)`. Tighten to a stable
substring like `/failed to create teacher/i` so a regression that
silently changes the toast copy is caught.

### 4. `as any` casts on Razorpay window stub

`BillingPage.test.tsx` lines 52-53, 63, 67. Setting `window.Razorpay`
in a test file legitimately needs an escape hatch; an interface
declaration `declare global { interface Window { Razorpay: ... } }`
would be more typesafe but adds boilerplate. Accept as-is for now.

## Positive Observations

- **Locale-/timezone-agnostic date assertion** (line 247: `/(?:Nov|Dec)\s+2026/`)
  — author noticed the IST/UTC boundary issue and matched both. Good.
- **Module-level service mocks via `vi.mock(...)`** — clean isolation,
  no leaked spies.
- **`importOriginal` spread for `useNavigate` / `useToast`** — preserves
  the rest of `react-router` / common-component exports rather than
  blanket-mocking the module.
- **`QueryClient` with `retry: false`** — keeps error-path tests fast
  and deterministic.
- **DRF field-error shape tested** for CreateTeacher
  (`{ email: ['A user with this email already exists.'] }`) — matches
  what the backend actually returns.
- **Cancel-button test for CreateTeacher** asserts the service was *not*
  called — correct negative assertion.
- **Empty-state coverage** for both pages (no plan, no invoices,
  required-field validation).

## Recommendations (non-blocking)

- Tighten currency regexes (item 1).
- Replace `document.querySelector('.animate-spin')` with a `data-testid`
  if the spinner ever gets refactored.
- Tighten `expect.any(String)` to a stable copy substring.
