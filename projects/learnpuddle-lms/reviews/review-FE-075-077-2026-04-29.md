### Anti-pattern scan (clean)
- No `it.skip` / `xit` / `it.todo`
- No `expect(true).toBe(true)` fake-pass
- No `as any` in test setup
- All `it()` blocks contain assertions

### Architecture claims verified
- **ProfilePage**: uses direct `mockedApi.patch` (no `QueryClientProvider` wrapper) — matches "no TanStack Query" claim ✅
- **AchievementsPage**: explicitly asserts `['studentGamification']` query key (line 506) — pins the cache key contract ✅
- **DemoBookingsPage**: inline `<select>` change tested via `userEvent.selectOptions` (preferred over `fireEvent.change`) ✅
- All three files apply `staleTime: Infinity + retry: false` (where QueryClient is in scope)

## Minor Issues (non-blocking)
- AchievementsPage at 49 tests for one page is on the heavy side. Consider whether some of the per-label assertions could be collapsed via a table-driven `it.each` to reduce test-runtime. Not blocking — the coverage is the asset.

## Positive Observations
- DemoBookingsPage covers BOTH modals (Create Booking + Send Email) including Cancel-without-call paths — a common gap in modal coverage.
- ProfilePage tests both avatar fallback (initials "AC" from "Alice Chen") AND `<img>` rendering when `profile_picture_url` is set — initials logic is famously bug-prone.
- AchievementsPage Retry button explicitly asserts `refetch` is called — verifies the error-recovery wiring.
- Coverage of singular-vs-plural label states ("1 day" vs "N days") guards against pluralization regressions.
