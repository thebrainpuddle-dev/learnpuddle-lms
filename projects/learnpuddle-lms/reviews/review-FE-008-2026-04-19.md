## Minor Issues

### m1 (polish) — HeadlessUI focus-trap lands on the close button

With the migration to `Dialog` / `DialogContent`, HeadlessUI's focus trap
auto-focuses the **first focusable element** on open. In `RubricPage.tsx`
the header renders the close `<button>` *before* the form, so opening
either Create or Edit Rubric lands focus on the close "X" rather than on
the Title `<input>`. Harmless but a small UX regression vs. the prior
hand-rolled modal where nothing was auto-focused.

Optional fix: add `autoFocus` to the title input, or use HeadlessUI's
`initialFocus` ref prop on the panel. Not a blocker.

### m2 (polish) — `deleteTitle` state never reset

`RubricPage.tsx:543-546` — `deleteTitle` is written only at click-time
(`setDeleteTitle(row.original.title)`) and never cleared. That's exactly
what enables the clean close-animation (the fix the review-request
called out), so clearing it would reintroduce the "undefined" flash. The
current behaviour is correct; flagging only because a future maintainer
reading the state list might be tempted to "clean it up" on close. A
one-line comment above the `useState` already documents this well
(`"during a re-render triggered by setDeleteTarget(null)"`). Good.

### m3 (polish) — SAML error banner uses structural `as`-cast for Axios

`SettingsPage.tsx:1202-1203`:

```tsx
const samlErrorStatus =
  (error as { response?: { status?: number } } | null)?.response?.status;
```

Works and is scoped to this card, but the codebase should eventually
adopt a shared `isAxiosError(err)` type guard (or import axios's own)
instead of structural casts. Low priority — the narrowing is strict
enough that a non-Axios error falls through to the generic "Failed to
load" branch without crashing. Matches the pattern used elsewhere in
this file (e.g. the mutation `onError` at L1129-1138).

### m4 (non-blocking) — Test claim `deleteTarget=null → undefined flash`

The test at `RubricPage.test.tsx:517-546` asserts that after Cancel, the
"Delete "Research Essay Rubric"?" text is **not in the document** — which
proves the dialog is fully gone, but does not explicitly assert the
**absence of a transient "Delete "undefined"?" flash** during the close
animation. In practice this is untestable via RTL because the close is
synchronous with React state but the animation is CSS-only, and RTL
doesn't tick through transitions. The test as written is sufficient
proof that the snapshot pattern is in place; the flash is ruled out by
inspection of lines 543-546 + 687-688 + 828-831. Not a blocker.

## Positive Observations

### RubricPage tests (FE-007 deferred)

- **32 tests, 0 regressions** — matches the request note (`Test Files 40
  passed / Tests 326 passed`). Covers every reviewer-requested area:
  pagination boundaries (hide on single page, disable Prev on p1, Next
  on last, page counter text, click → page=2 query), debounce (300 ms
  gate + search resets page, Clear button), deleteTitle snapshot
  (captured at click-time, survives unmount), modal open/close, clone
  mutation + cache invalidation, delete confirm/cancel, error state
  (no crash on reject).
- **Debounce strategy is correct**: real timers + `waitFor` with
  `timeout: 1500` ms. Fake timers with async userEvent are the classic
  footgun here — the chosen approach avoids it. The test note at
  L313-314 explicitly documents this choice, which is the kind of thing
  future maintainers will thank someone for.
- **DataTable stub** is pragmatic — calls each column's `cell`
  renderer so the row action buttons actually mount, without pulling in
  TanStack Table's real implementation. Falls back to plain-text for
  plain-string columns. Scoped to the test file; no leak risk.
- **`makeRubric` fixture factory + `makeListResponse`** with
  `next`/`previous` URLs matching the real paginator shape — tests will
  surface pagination-shape regressions, not just react-query mechanics.
- **Error-state test** rejects `listRubrics` with a network error and
  asserts the heading still renders — proves the page doesn't crash on
  API failure.
- Every async assertion uses `waitFor` / `findBy*` — no flaky
  `setTimeout` waits, no manual `act()` spamming.

### SAML error banner (FE-006 m4)

- `retry: false` on the `useQuery` — crucial for feature-gated endpoints
  that return 403 permanently; otherwise the user sees multiple identical
  403s in the network tab before the banner appears. Good.
- **403 vs. generic error is surfaced explicitly** — "SAML SSO is not
  enabled for this school" vs. "Failed to load SAML configuration" —
  which tells admins on non-enterprise plans the *why*, not just the
  *what*. Copy is appropriate for the audience.
- **Card header is preserved on the error path** — admins know *which*
  card errored (there are three on the Security tab); dropping the
  header would make it ambiguous. Thoughtful.
- `ExclamationTriangleIcon` and `ShieldCheckIcon` both `aria-hidden` —
  the red text is the accessible signal, the icon is decorative. Correct.
- No new dependencies; reuses icons already imported at the top of the
  file. Zero bundle impact.

### RubricModal a11y (FE-007 m3)

- **Focus trap, Escape-to-close, `role="dialog"`, `aria-modal`** — all
  inherited from HeadlessUI's `Dialog` via the existing
  `components/ui/dialog.tsx` wrapper. This is exactly the right move:
  reuse the project's existing primitive rather than re-implementing.
- **Layout preserved** with `showClose={false}` + className override
  (`p-0 flex flex-col` + `overflow-hidden`). TailwindMerge resolves
  against the wrapper's default (`p-6 overflow-y-auto sm:max-w-lg`) so
  the custom header + scrollable body + sticky footer all work.
- **`DialogTitle` replaces the hand-rolled `<h2>`** — HeadlessUI wires
  `aria-labelledby` automatically, so screen readers announce "Create
  Rubric" / "Edit Rubric" on open without any manual aria-wiring.
- The inner form still uses `flex flex-col flex-1 min-h-0` so the
  scrollable body + sticky footer stay inside the DialogPanel. Correct.
- Comment at L386-387 documents *what* HeadlessUI is providing — future
  readers won't wonder "wait, where's the focus trap?"
- **`onOpenChange={(open) => { if (!open) onClose(); }}`** — correct
  wiring: HeadlessUI's `onClose` is called for Escape, backdrop click,
  and outside-click; this adapter funnels all of them to the existing
  `onClose` handler, so the dialog behaves consistently regardless of
  which dismiss path the user takes.
