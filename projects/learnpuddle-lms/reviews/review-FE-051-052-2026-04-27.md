# Review: FE-051 (MyCoursesPage) + FE-052 (AssignmentsPage) — Teacher Page Tests

## Verdict: APPROVE

## Summary

52 new tests on the teacher-facing course list and assignments pages.
Both verified green individually (22/22 + 30/30). Author flagged
intermittent flakes in pre-existing `RubricPage.test.tsx` and
`maicActionEngine.audioCache.test.ts` — those are out of scope here and
predate this work.

## Critical Issues

None.

## Major Issues

None.

## Minor Issues

### 1. `AssignmentsPage` — multiple "Submit" buttons disambiguated by index

Line ~354 / ~477: `getAllByRole('button', { name: /^submit$/i })[0]`
and `[confirmButtons.length - 1]`. The author documented this pattern
because both the page-header and the modal have a "Submit" button.
Works but is order-dependent; adding `data-testid="modal-submit"` on
the modal confirm button would be more robust. Style nit.

### 2. Loose `.length >= 1` checks for status badges

`MyCoursesPage.test.tsx` lines ~166-190 and `AssignmentsPage.test.tsx`
lines ~252-272 use `getAllByText().length >= 1` for badges that appear
in both filter pills and course/assignment cards. The pattern is
defensible (the same status string legitimately appears in multiple
DOM nodes), but `getAllByText('Completed').length === 2` (or whatever
the deterministic count is in the fixture) would be tighter.

## Positive Observations

- **`SubmissionModal` stubbed at the boundary** with a close-button
  spy — tests verify modal lifecycle (`isOpen`, `onClose`) without
  re-testing the modal's internals. That component has its own tests.
- **Quiz vs. text-assignment branching** is explicitly covered: text
  → opens textarea + submit flow; quiz → calls `navigate('/teacher/...')`.
  This is the exact kind of conditional that bites in production.
- **Empty-state variants** — "No assessments found" tested per tab
  (no assignments overall, no graded, etc.).
- **Score display** (`42/50` plus the `Score` label) tested for the
  GRADED state.
- **Status-tab filtering** drives the mock with
  `listAssignments(status?)` — exercises the server-driven filter path.
- **Mutation success + error paths** — submit success toast, submit
  error toast, cancel path (no service call) all covered.
- **Filter regex `/^Completed\d+$/`** to disambiguate the filter pill
  ("Completed3") from a course-card status ("Completed") — author
  documented the disambiguation.

## Recommendations (non-blocking)

- Add `data-testid` on the modal submit button to remove the
  index-based selector.
- Tighten `length >= 1` to exact counts where the fixture is
  deterministic.
