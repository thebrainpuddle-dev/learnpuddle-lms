## Minor Issues

1. **CSS-class selectors couple tests to styling.** Recurring pattern:
   - `document.querySelectorAll('.tp-skeleton')` — DiscussionPage L169, SectionDashboardPage L278, DiscussionThreadPage L178
   - `document.querySelector('.rounded-xl.bg-gray-50.border')` — DiscussionPage L319, L336
   - `document.querySelector('.animate-pulse')` — ProfessionalGrowthPage L193
   - `document.querySelector('form')` and `form.querySelector('button[type="submit"]')` — DiscussionThreadPage L329-331, L340-341, L351-352

   If a designer renames `tp-skeleton` to `lp-skeleton` or refactors the filter container, these
   tests break for cosmetic reasons. **Recommendation (non-blocking):** add `data-testid="skeleton"`
   on skeletons and `data-testid="status-filter-bar"` / `data-testid="reply-form"` to anchor
   structural queries. Acceptable now because (a) consistent with established teacher-page test
   patterns, (b) the trade-off is small for 1-line queries, (c) the wider repo accepts class-based
   queries elsewhere.

2. **MAICCreatePage back button identified by index, not label.** `getAllByRole('button')[0]`
   (L79-81). Brittle if the component layout changes. Root cause is the component itself: the back
   button has an SVG icon with no `aria-label` or `<title>`, so it has no accessible name. **This
   is a production a11y issue** — the icon-only back button is invisible to screen readers
   (`MAICCreatePage.tsx` L37-42). Flagging for the frontend team (file as a separate FE ticket,
   not blocking these test PRs since they're test-only). Test workaround is acceptable for now.

3. **`screen.getByText('5')` / `'12'` / `'23'` / `'47'` for reply/view counts** (DiscussionPage
   L218-219, L226-227). Numeric text matchers can collide if other UI elements happen to render
   the same number (e.g., a future "5 unread" indicator). Same caveat for `'2'`, `'30'`, `'22'`
   in SectionDashboardPage analytics (L485-486) and ProfessionalGrowthPage stats. Use
   `within(card)` scoping where ambiguity is plausible. Non-blocking — fixtures are deliberately
   chosen to avoid current collisions.

4. **DiscussionThreadPage hide-confirm assertion uses `expect.stringContaining('/moderate/')`**
   (L437). Loses URL specificity — would still pass if the URL silently changed to
   `/v1/teacher/discussions/threads/thread-1/replies/reply-2/moderate/something-wrong/`. Tighten
   to a full path or `expect.stringMatching(/\/replies\/reply-1\/moderate\/$/)`.

5. **SectionDashboardPage "shows student count badge"** (L318-326) asserts `studentsTab.textContent`
   matches `/2/`. Will pass for any digit "2" anywhere in the tab — including ARIA labels or icon
   counters elsewhere. Tightening to a scoped role/text query would be sturdier. Non-blocking.

## Positive Observations

- **Per-page service mocks** (`academicsService.getSectionDashboard`, `teacherService.getCompetencyDashboard`,
  `gamificationService.*`) are at the right layer — tests don't need to know HTTP details, but
  individual mutations still verify exact REST paths (`/v1/teacher/discussions/threads/thread-1/subscribe/`,
  `/v1/teacher/discussions/threads/thread-1/replies/`). Best of both worlds.
- **`importOriginal` spread for `useNavigate`** is consistent across all 5 files and avoids the
  classic `Cannot find element` error from over-broad react-router-dom mocks.
- **DiscussionThreadPage moderation tests** correctly cover both visibility (`can_moderate=true`
  shows controls / `false` hides them) and behavior (PATCH/POST payloads). The `/^pin$/i`
  anchored regex (L382, L403) is a deliberate guard against matching "Pinned"/"Unpin" — good
  attention to detail.
- **ProfessionalGrowthPage `getAllByText('Growth area')` length=2** (L298-300) is the right shape
  for asserting filter count — a single `getByText` would have crashed on the duplicate.
- **SectionDashboardPage tab switching** via real `userEvent.click` on the `tablist` exercises
  actual TabGroup state and panel mounting/unmounting, not stubbed switching. High-fidelity.
- **DiscussionThreadPage closed-thread reply form hidden** (L315-322) catches a real
  authorization-style branch that often gets missed.
- **Empty-state coverage is uniformly strong**: "No discussions yet", "No students in this
  section", "No courses targeting this section", "No assignments for this section",
  "No Skills Assigned Yet", "No replies yet.", "Thread not found" — every page tests its
  zero-state copy.
- **`gcTime: 0, retry: false` per-test QueryClient** is set everywhere; no risk of cross-test
  cache pollution.
- **`makeThread(overrides)` helper in DiscussionThreadPage** lets each test express only what it
  cares about (`{ can_moderate: true }`, `{ status: 'closed' }`, `{ is_subscribed: true }`)
  while sharing a default body. Idiomatic and DRY.
