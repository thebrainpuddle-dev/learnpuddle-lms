# Review: FE-029 — RubricPage test flake fix (clearAllMocks → resetAllMocks)

## Verdict: APPROVE

## Summary
One-line test-only fix that converts `vi.clearAllMocks()` to `vi.resetAllMocks()` in
`RubricPage.test.tsx`'s `beforeEach`. Root cause (Vitest's `clearAllMocks` only wipes call-history
but leaves `.mockResolvedValue()` implementations and `.mockResolvedValueOnce()` queues intact) is
correct and well documented inline. Full suite now deterministic at 557/557.

## Critical Issues
None.

## Major Issues
None.

## Minor Issues

1. **The inline comment at L218-225** is excellent but long (7 lines). Consider shortening to the
   essential sentence — "resetAllMocks wipes implementations and mockResolvedValueOnce queues;
   clearAllMocks only wipes call-history" — and leaving the full failure-mode explanation in the
   FE-029 coordination note for future spelunkers. Non-blocking.

2. **This is a suite-wide pattern problem.** A quick grep would find other files using
   `vi.clearAllMocks()` in `beforeEach` after queuing `mockResolvedValueOnce(...)` in individual
   tests — they're latent flakes. Adding an ESLint rule (or the existing "usefaketimers" lint-rule
   pattern from FE-LINT-RULE-USEFAKETIMERS) for "prefer `resetAllMocks` over `clearAllMocks` in
   `beforeEach`" would be a good follow-up ticket. Not in scope for FE-029.

## Positive Observations

- **Diagnosis is precise.** The "pending React Query microtask consumes the queue entry before the
  next test's initial-render API call" explanation matches the observed symptom (Next button enabled
  when it should be disabled because `count` came back as 25 instead of 12) and is the kind of
  detail that only a careful reader of Vitest + React Query semantics would land on.
- **Minimal surface area.** One line changed, `setupDefaultMocks()` immediately re-establishes every
  default implementation so no test loses coverage.
- **Correct ordering.** `vi.resetAllMocks()` → `setupDefaultMocks()` in that order — reset first
  ensures a clean slate, setup re-installs the baseline. Reversing would leak the previous test's
  overrides.
- **RubricPage.test.tsx is a large, high-value test file (32 tests)** — fixing its flake keeps CI
  green and stops the "re-run it, probably passes" normalization that erodes test trust.

## Scope
Test-only, zero production impact, zero risk.
