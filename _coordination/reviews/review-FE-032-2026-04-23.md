## Minor Issues

- **Zod regex drifts from the review request** (non-blocking).
  Request doc cites: `/^[\w\s\-]+$/` with message "Only letters, numbers,
  spaces, hyphens, and underscores". Actual code (`SettingsPage.tsx:1562`)
  is: `/^[\w\s\-()]+$/` with message "Only letters, numbers, spaces, hyphens,
  and parentheses are allowed". The regex change is fine (admins want names
  like `Okta (Prod)`), but the **error message no longer mentions underscores
  even though `\w` allows them**. Tighten the message to: *"Only letters,
  numbers, spaces, hyphens, underscores, and parentheses"*. The QA test
  `rejects token names with special characters outside the allowed set` will
  still pass as long as it doesn't submit an underscore, which is already
  allowed â€” but verify QA's fixture doesn't include an underscore in the
  negative case.

- **`SCIMTokenCard` is not exported from `SettingsPage.tsx`** â€” QA flagged
  this in their test-request doc. Not a blocker since the QA tests render the
  full page at `?tab=security`, which exercises real wiring. But if a future
  consumer needs isolated mounting (storybook, docs), exporting the component
  should be trivial. No change required now.

- **Revoke modal closes before mutation settles** (advisory).
  `onConfirm` calls `revokeMutation.mutate()` and then `setRevokeTarget(null)`
  synchronously. This means the `loading={revokeMutation.isPending}` prop
  on `ConfirmDialog` will never render truthy because the dialog is already
  closed when the mutation starts. If the intent is to show a loading spinner
  in the dialog while revoking, move the close into `onSuccess`/`onSettled`
  of the mutation. If intent is optimistic dismissal (accept and trust the
  toast to report the outcome), this is fine â€” just note that the `loading`
  prop is dead code in the current shape.

## Positive Observations

- **One-time-reveal pattern is correct.** Raw token is held only in local
  state (`revealToken`) and never re-fetched. Modal close clears the state,
  and the list query invalidation refreshes only the summary view (which
  has no raw token field by contract). Backend hashes with SHA-256; this
  frontend matches the "cannot recover" guarantee.
- **Clipboard UX is solid.** `navigator.clipboard.writeText` wrapped in a
  `.then()` that flips a visual indicator and surfaces a `toast.success`
  with explicit next-step instruction. `select-all` class on the `<code>`
  block allows triple-click fallback if clipboard API is blocked.
- **Destructive action guarded.** Revoke routes through `ConfirmDialog` with
  a contextual message including the token name and the revocation
  consequence. `variant="danger"` ensures the confirm button is styled
  red. Cancel label "Keep active" is the right positive framing for a
  revoke confirmation.
- **Error handling.** Both mutations have `onError` handlers that surface
  a `toast.error` with a sensible fallback message when the Error has no
  `.message`.
- **Loading + error + empty states all rendered.** The table branch is only
  hit when `!isLoading && !isError && tokens.length > 0`, preventing ghost
  rows during transitions.
- **Subdomain fallback placeholder** (`https://<your-school>.learnpuddle.com/scim/v2/`)
  is a nice UX touch for admins viewing the page before a tenant theme loads.
- **A11y**: `aria-label="Copy token to clipboard"` on the copy button,
  `focus-visible:ring-*` on the revoke link, `role` semantics via Headless
  UI Dialog + `Dialog.Title`.
- **TanStack Query cache keys** (`['scim-tokens']`) are stable and shared
  across list/create/revoke invalidations.
- **Type discipline**: `SCIMTokenSummary` (no raw token) and
  `SCIMTokenCreated` (with raw token) are correctly separated. Impossible
  to accidentally render the raw token in the list.
