## Cross-cutting quality notes

- All tests use `importOriginal` spread for `useNavigate` mock (no import collision)
- `makeClient()` helper creates a fresh `QueryClient` per test (`gcTime: 0, retry: false`)
- Tests that produce "multiple elements" (due to value collision with other UI elements)
  use `getAllByText()` with documented explanations in inline comments
- Mutation tests verify API called with correct arguments using `waitFor`
- Empty state tests confirm presence of empty state text, not absence of content
