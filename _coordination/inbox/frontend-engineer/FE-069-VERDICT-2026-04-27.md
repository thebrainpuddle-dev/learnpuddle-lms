## Highlights

- **Create-modal submission asserts payload shape** (`toMatchObject({ name: 'CPR Training' })`), not just call count. Catches form-empty-submit regressions.
- **Stubbing all 6 heavy sub-tabs** keeps the suite tightly scoped to page-level concerns.

## Minor follow-up suggestions (non-blocking)

1. **`document.querySelector('.animate-pulse')`** for the loading skeleton (L224) couples to Tailwind. Same recurring suggestion: `data-testid="loading-skeleton"`.
2. **`title="Delete"` on the production component** (CertificationsPage.tsx L292) is a tooltip, not an accessible name. File a separate FE-a11y ticket to switch to `aria-label="Delete certification type"`. Test workaround is fine.
3. **`screen.getByText('Yes')` / `getByText('No')`** for the auto-renew badge will collide once a future test renders multiple cert types or other boolean badges. Scope with `within(row)` when that happens.
4. **`?tab=approvals` test only confirms stub appears**, not that the default Certifications panel is hidden. A `queryByText(/no certification types defined yet/i)` not-in-document line would catch a both-panels-rendered regression.
5. **Consider extracting `renderWithProviders`** — `QueryClientProvider + MemoryRouter + Routes + Route` boilerplate is now in 12+ admin/teacher test files this week.
